## Version Control Systems

Link your Unseen workspace to GitHub or GitLab to automatically pull changes into your documentation.

<Tabs>
  <Tab title="GitHub" icon="github">
    <Steps>
      <Step title="Create Webhook" icon="link">
        In your GitHub repo settings, add a webhook pointing to `https://api.example.com/v1/webhooks/github`.
      </Step>
      <Step title="Configure Events">
        Select events like pushes and pull requests.
      </Step>
      <Step title="Add Secret">
        Use a secret like `YOUR_WEBHOOK_SECRET` for verification.
      </Step>
    </Steps>
  </Tab>
  <Tab title="GitLab" icon="package">
    <Steps>
      <Step title="Set Webhook URL">
        Enter `https://api.example.com/v1/webhooks/gitlab`.
      </Step>
      <Step title="Enable Triggers">
        Choose merge request and push events.
      </Step>
      <Step title="Test Connection">
        Push a commit to verify sync.
      </Step>
    </Steps>
  </Tab>
</Tabs>

<Callout kind="tip">
  Enable two-way sync in Unseen settings to push changes back to your repo.
</Callout>

## Webhook Setups

Use webhooks to automate workflows. Unseen sends events to your endpoint on doc updates.

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  const express = require('express');
  const app = express();
  app.use(express.json());

  app.post('/webhook', (req, res) => {
    if (req.body.event === 'doc_updated') {
      console.log('Doc changed:', req.body.data);
    }
    res.status(200).send('OK');
  });

  app.listen(3000);
  ```
  ```python
  from flask import Flask, request

  app = Flask(__name__)

  @app.route('/webhook', methods=['POST'])
  def webhook():
      if request.json['event'] == 'doc_updated':
          print('Doc changed:', request.json['data'])
      return 'OK', 200

  if __name__ == '__main__':
      app.run(port=3000)
  ```
</CodeGroup>

Configure your webhook in Unseen dashboard at `https://dashboard.example.com/settings/webhooks`.

## API Access for Custom Extensions

Build bespoke integrations using Unseen's REST API.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}`.
</ParamField>

<ParamField query="workspace_id" param-type="string" required="true">
  Your Unseen workspace identifier.
</ParamField>

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X POST https://api.example.com/v1/docs \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"title": "New Doc", "content": "Hello world"}'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      title: 'New Doc',
      content: 'Hello world'
    })
  });
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "id": "doc_123",
    "title": "New Doc",
    "status": "success"
  }
  ```
</Response>

## Syncing with Project Management Tools

Connect Unseen to tools like Jira or Linear for ticket-linked docs.

<Expandable title="Jira Integration" default-open="true">
  1. Generate API token in Jira.
  2. In Unseen, add `https://your-jira-instance.atlassian.net` and `{YOUR_JIRA_TOKEN}`.
  3. Link docs to tickets using `#JIRA-123` syntax.
</Expandable>

<Expandable title="Linear Sync">
  Use webhooks to create issues from doc comments.
</Expandable>
